###怎么使用protobuf
```
编写统一的proto文件，编译生成不同语言的文件;
不同语言下，使用同一套proto文件
```
###为什么使用protobuf
```
跨平台: 
    如果不使用protobuf，在不同平台下，java客户端发送数据，c++服务端中接收数据，如何保证数据在内存上的一致性？比如：
    c++的结构体有内存对齐，但是java没有；c++的char为1个字节，java的char为2个字节等。
序列化：
    用于编解码，或者网络数据传输（二进制传输），序列化后数据更小，解析更快，非明文传输。
反射：  
    在没有提供proto文件及对应h和cpp文件下，通过反射可以拿到所有数据;
    可以用在protobuf转json，或者打印protobuf中的信息。（如果不适用反射，如果要打印protobuf中的信息，需要通过set_name和name等方法来实现，
```
